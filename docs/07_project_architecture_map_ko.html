AWS Serverless Portfolio 전체 구조 지도

현재 프로젝트가 프론트엔드, 백엔드 API, AWS 호스팅, V2 로컬 AI 챗봇까지 어떻게 이어지는지 한눈에 보기 위한 문서다.

한 줄 요약

React 정적 사이트는 S3에 저장되고 CloudFront/Route 53을 통해 shinseong.dev로 제공된다. 화면은 API Gateway를 호출하고, 일반 프로필 API는 Lambda에서 바로 응답한다. 챗봇은 AWS가 질문을 SQS에 넣고, MacBook의 local agent가 Docker 안의 llama.cpp 모델로 답변을 만들어 DynamoDB에 저장하는 구조다.

핵심 주소와 위치

Website https://shinseong.dev
API Gateway ca-central-1 / Prod
Frontend Hosting S3 private + CloudFront OAC
Chat Status currently disabled

프론트엔드 구조

역할

frontend/는 Vite + React + TypeScript 정적 앱이다. 빌드하면 HTML/CSS/JS 정적 파일이 되고, 이 파일들이 S3에 올라간다.

탭은 Home, Projects, Resume, AI Roadmap, AI Chat으로 구성된다.

주요 파일

frontend/src/App.tsx 화면 탭, API 상태, 채팅 UI
frontend/src/api.ts /health, /profile 호출
frontend/src/chatApi.ts POST /chat, GET /chat/{requestId} polling
frontend/public/chat-config.json 채팅 UI 활성/비활성 설정
frontend/src/styles.css 전체 UI 스타일

백엔드 API 구조

V1 Portfolio API

PortfolioApiFunction은 기본 포트폴리오 데이터를 제공한다. API 연결 상태와 프로필/프로젝트/자격증 정보를 React 앱에 전달한다.

GET /health GET /profile OPTIONS

V2 Local AI API

LocalAiFunction은 채팅 요청을 직접 모델에 보내지 않는다. 입력을 검증하고 DynamoDB에 PENDING 상태를 저장한 뒤 SQS에 job을 넣는다.

POST /chat GET /chat/{requestId} OPTIONS

챗봇 처리 흐름

1. React AI Chat

사용자가 질문 입력. postChat()이 POST /chat 호출.

→

2. LocalAiFunction

입력 검증 후 DynamoDB에 PENDING 저장, SQS에 작업 발행.

→

3. MacBook Agent

sqs_agent.py가 SQS long polling으로 질문을 가져감.

→

4. Docker Model

llama.cpp 서버의 /v1/chat/completions로 로컬 추론.

답변 생성 후 agent가 DynamoDB 항목을 DONE 또는 ERROR로 업데이트한다. React는 GET /chat/{requestId}를 반복 호출해서 결과를 가져온다.

AWS 리소스 지도

영역 리소스 리전 역할
도메인 Route 53 Hosted Zone Global shinseong.dev를 CloudFront로 연결
HTTPS 인증서 ACM Certificate us-east-1 CloudFront에 붙는 TLS 인증서. CloudFront용 ACM은 us-east-1 필요.
정적 호스팅 CloudFront Distribution Global HTTPS, 캐시, S3 origin 보호 접근
정적 파일 S3 PortfolioBucket us-east-1 React 빌드 결과물 저장. Public access 차단.
API API Gateway ca-central-1 프론트에서 Lambda를 호출하는 HTTP 엔드포인트
Portfolio API PortfolioApiFunction ca-central-1 /health, /profile 응답
Chat API LocalAiFunction ca-central-1 채팅 요청 접수, 상태 조회, SQS enqueue
Queue chat-jobs ca-central-1 MacBook local agent가 가져갈 질문 job 저장
State chat-requests ca-central-1 채팅 요청 상태와 답변 저장. TTL 3600초.
Logs CloudWatch Logs ca-central-1 Lambda 로그. Local AI 함수는 14일 보관.

http://127.0.0.1:8080/v1/chat/completions 호출

보안 의도

모델은 Docker 컨테이너 안에서만 실행된다. 앱은 localhost:8080으로만 통신한다.

oMLX/pi/Hermes 경로를 사용하지 않는다. Docker socket, AWS credential, SSH key, Home directory는 컨테이너에 마운트하지 않는다.

모델 context는 local_ai/harness/context/와 local_ai/harness/prompts/system_prompt.md를 기반으로 제한한다.

환경변수 사용 위치 의미

현재 운영 상태

AWS 리소스는 배포되어 있지만, 공개 채팅은 안전하게 꺼진 상태다. frontend/public/chat-config.json의 enabled가 false이고, Lambda 환경변수 CHATBOT_ENABLED도 false다.

즉, 사이트와 API는 배포되어 있지만 방문자가 실제 로컬 모델로 질문을 보내는 단계는 아직 열려 있지 않다.

핵심 체크포인트

S3는 private CloudFront OAC만 S3 읽기 가능 API는 ca-central-1 CloudFront 인증서는 us-east-1 채팅은 SQS 비동기 방식 MacBook에 inbound port 필요 없음 현재 public chat disabled
